## Prerequisites

Before you begin, make sure you have the following:

- Node.js 18 or later
- npm 9 or later
- A Firebase project with Authentication and Firestore enabled
- A valid Mobility 4 API bearer token

## Set up the application

<Steps>
  <Step title="Clone the repository">
    Clone the Mobility 4 source repository to your local machine.

    ```bash
    git clone https://github.com/your-org/mobility4.git
    cd mobility4
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required npm packages.

    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in your values.

    ```bash
    cp .env.example .env
    ```

    Open `.env` and set the following variables:

    ```ini .env
    VITE_API_URL=https://mobility-app.com/api/
    VITE_ACTIVE_TOKEN=
    VITE_GOOGLE_MAPS_API_KEY=
    VITE_FIREBASE_API_KEY=
    VITE_FIREBASE_AUTH_DOMAIN=
    VITE_FIREBASE_PROJECT_ID=
    VITE_FIREBASE_STORAGE_BUCKET=
    VITE_FIREBASE_MESSAGING_SENDER_ID=
    VITE_FIREBASE_APP_ID=
    VITE_FIREBASE_MEASUREMENT_ID=
    ```

    <Note>
      `VITE_API_URL` defaults to `https://mobility-app.com/api/` if left blank. Set `VITE_ACTIVE_TOKEN` to the Bearer token used to authorize all REST API calls. See [Authentication](/authentication) for details on obtaining this token.
    </Note>
  </Step>

  <Step title="Start the development server">
    Run the Vite development server.

    ```bash
    npm run dev
    ```

    The application opens at `http://localhost:5173` by default.

    <Tip>
      Vite hot-reloads on every file save, so you can keep the server running while you explore the codebase.
    </Tip>
  </Step>
</Steps>

## Complete your first field operation

Once the application is running, follow these steps to go from sign-in to tracking your first field activity.

<Steps>
  <Step title="Sign in">
    Navigate to `http://localhost:5173/auth/sign-in`. Enter your **Email or ID number (cédula)** and **Password**, then click **Iniciar Sesión**.

    You can sign in with either your registered email address or your numeric ID number (7–8 digits). After a successful login, you are redirected to the **Dashboard**.

    <Tip>
      If you have forgotten your password, click **¿Olvidaste tu contraseña?** on the sign-in screen to go to `/auth/password-reset`.
    </Tip>
  </Step>

  <Step title="Navigate to the dashboard">
    After sign-in, you land on the **Ecommerce** dashboard at `/dashboard/ecommerce`. This page gives you an overview of field activity metrics across your assigned clients and regions.
  </Step>

  <Step title="Create a user">
    Go to **Users** (`/User`) using the sidebar navigation. Click **Create User** and fill in the required fields: name, email, ID number, and password. Complete the configuration, security, regions, and modules steps in the creation wizard.

    <Note>
      User creation is multi-step. Each step validates its fields before allowing you to proceed. Make sure you assign at least one client and one region in the **Clientes** and **Regiones** steps.
    </Note>
  </Step>

  <Step title="Assign the user to a region">
    Once the user is created, navigate to **User Assignment** (`/AsignacionUsuarios`). Select the target region and assign the newly created user. This step links the user to a route and region so they appear in field tracking filters.
  </Step>

  <Step title="Track field activity">
    Navigate to **Seguimiento** (`/seguimiento`) in the sidebar. Select a **Client** and a **Region** from the filter dropdowns — these options are populated from your own `userData` in localStorage. Then select a **User** (or choose **Todos los usuarios** to see all) and a date, and click **Buscar** to load GPS coordinates on the interactive map.

    <Tip>
      The map updates in real time based on the selected filters. Use the date picker to review historical field activity for any given day.
    </Tip>
  </Step>
</Steps>
