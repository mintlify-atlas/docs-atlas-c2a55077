The Mobility 4 API is a REST API that gives you programmatic access to field operations data: clients, users, roles, regions, establishments, routes, inventory, orders, forms, and real-time tracking. All requests and responses use JSON.

## Base URL

All API paths are relative to:

```
https://mobility-app.com/api/
```

You can override the base URL at build time using the `VITE_API_URL` environment variable.

## Authentication

Every request must include a Bearer token in the `Authorization` header:

```
Authorization: Bearer <token>
```

The token is read from the `VITE_ACTIVE_TOKEN` environment variable. See [Authentication](/api/authentication) for details.

## Content type

All requests must send and accept JSON:

```
Content-Type: application/json
```

## Environment variables

Configure the API URL and token in your `.env` file:

```bash .env
VITE_API_URL=https://mobility-app.com/api/
VITE_ACTIVE_TOKEN=your_bearer_token_here
```

These values are consumed by `src/Api/enviroment.js`:

```javascript src/Api/enviroment.js
const settings = {
  API_URL: import.meta.env.VITE_API_URL || "https://mobility-app.com/api/",
  ACTIVE_TOKEN: import.meta.env.VITE_ACTIVE_TOKEN || "",
};

export default settings;
```

## Request and response format

Request bodies are JSON objects sent in the request body for `POST` and `PUT` methods. Responses are JSON objects or arrays returned directly from `response.data`.
