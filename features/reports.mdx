The **Reports** page at `/reportes` lets you query, filter, and export data collected by field agents across your client portfolio. You select the client, form type, region, and date range, then generate a downloadable file containing the matching records.

## Filtering a report

The report form has five fields:

| Field | Description |
|---|---|
| **Cliente** | The client whose data you want to query. Selecting a client loads the available forms for that client. |
| **Formulario** | The form name to filter results by. Options populate automatically after you select a client. |
| **RegiĂłn** | A specific region to narrow results, or **TODAS LAS REGIONES** to include all regions. |
| **Desde** | Start date of the report range (defaults to 7 days ago). |
| **Hasta** | End date of the report range (defaults to today). |

<Note>
The **Formulario** dropdown is empty until you select a **Cliente**. The platform fetches available forms from `POST /formularios/getFormsWeb` using the selected client name.
</Note>

## Generating a report

1. Select a **Cliente** from the dropdown. The platform loads that client's available forms.
2. Select a **Formulario**.
3. Select a **RegiĂłn**, or leave it set to **TODAS LAS REGIONES**.
4. Set the **Desde** and **Hasta** dates.
5. Click **Generar reporte**.

The platform calls `POST /analytics/generateReportAnalytics/generateReport` with the selected parameters. If records exist for the date range, a CSV file downloads automatically.

<Warning>
If no records exist for the selected filters and date range, the platform shows a warning: "No se encontraron registros en la fecha dada." Try widening the date range or selecting a different region.
</Warning>

## Fundamental form behavior

When you select a Fundamental-type form (any variation of `Fundamental`, `Fundamentales`, or `FUNDAMENTAL`), the platform splits the results into two separate download files:

- **Fundamental** â€” all responses where `pregunta` is not a price field
- **Precios** â€” responses where `pregunta` is `PRECIO`, `BOLIVARES`, or `DOLARES`, normalized to a single `PRECIO` column with a `moneda` field (`BS` or `$`)

Both files download in sequence automatically.

## Data columns

Standard report files include the following columns:

```text
formulario, fecha_Formulario, fecha_Sincronizado, region, usuario,
razon_social, canal, ciudad, cod_est, establecimiento, categoria,
subcategoria, marcaProducto, codigo, producto, pregunta, respuesta,
moneda, tasa
```
